## 検証ポイント

以下の点から計画を評価してください:

1. 網羅性: すべての必要なステップが含まれているか
   - タスクの目標達成に必要な要素がすべて含まれているか
   - 重要な前提条件や準備作業が考慮されているか
   - 成果物の品質確保のためのレビューや検証ステップが含まれているか

2. 依存関係: サブタスク間の依存関係は適切か
   - 依存関係のサイクルや矛盾がないか
   - 並行実行可能なタスクが適切に識別されているか
   - クリティカルパスが明確か

3. 具体性: 各サブタスクは十分に具体的で実行可能か
   - タスクの目的と期待される成果物が明確か
   - 必要なリソースと制約が特定されているか
   - 成功基準が明確に定義されているか

4. 効率性: 実行順序は効率的か
   - タスクの順序は論理的で効率的か
   - リソースの活用が最適化されているか
   - 不必要な待機時間や重複作業がないか

5. 一貫性: タスク全体の目標との整合性はあるか
   - すべてのサブタスクが全体の目標に貢献しているか
   - 技術的な制約や要件が考慮されているか
   - 品質基準や期待値が明確に反映されているか

## 出力形式

以下のJSON形式で検証結果を出力してください:

```json
{
  "is_valid": true/false,
  "issues": [
    "問題点1の説明",
    "問題点2の説明"
  ],
  "suggestions": [
    "改善提案1",
    "改善提案2"
  ],
  "analysis": {
    "completeness_score": 0-100,
    "dependency_score": 0-100,
    "specificity_score": 0-100,
    "efficiency_score": 0-100,
    "consistency_score": 0-100,
    "overall_assessment": "全体的な評価と分析",
    "risk_factors": [
      "潜在的なリスク1",
      "潜在的なリスク2"
    ],
    "improvement_priorities": [
      "優先的に改善すべき項目1",
      "優先的に改善すべき項目2"
    ]
  }
}
```

計画を検証し、詳細な分析結果を提供してください。各スコアは以下の基準で評価してください:

- 90-100: 優れている
- 70-89: 良好
- 50-69: 改善の余地あり
- 0-49: 重大な問題あり
